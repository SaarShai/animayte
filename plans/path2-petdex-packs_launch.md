## License hygiene (important — read first)
The Petdex catalog (`https://github.com/crafter-station/petdex`, API `https://petdex.crafter.run/api/manifest`) contains 2700+ pets, **many of undeclared / fan-art license**. The *format/contract* is safe to adopt; the *pet assets are not safe to redistribute*. Therefore:
- **Fetch and render Petdex pets ON DEMAND** (client-side, at runtime). **Do NOT commit downloaded third-party spritesheets** into the repo.
- The only pet asset you commit is the **license-clean `pets/codex-demo`** synthetic fixture (already generated by `tools/make-codex-fixture.mjs`).
- Add a clear `pets/petdex-picker/README.md` (or header) stating the license caveat and that pets are loaded from their source, not bundled.
- If unsure about any redistribution question, **ask Saar.**

## Current state (grounded)

## Build it (phases, each with a verification gate)
1. **Commit the bridge** — add the four `lib/codex/*` modules + `pets/codex-demo` + `test/codex.test.mjs` + `tools/make-codex-fixture.mjs`. *Gate:* `node test/codex.test.mjs` green; `node tools/make-codex-fixture.mjs` reproduces the fixture.
2. **Play the demo pet live** — serve the repo (`node grid/serve.mjs 4370` or `bin/animayte start`), load the demo Codex pet through `createCodexRuntime`, drive `setMood`/`reactByName`/`setFullness`. *Gate:* screenshot of the demo pet animating its states.
3. **Petdex catalog loader** — `lib/codex/petdex-loader.mjs`: fetch `https://petdex.crafter.run/api/manifest`, normalize to `{ id, name, manifestUrl }[]`, cache in memory/localStorage. *Gate:* logs N≈2700 pets; handles offline/error gracefully.
4. **Picker UI** — `pets/petdex-picker/index.html`: a gallery of catalog entries; clicking one loads it via `createCodexRuntime({ manifestUrl })` into a preview canvas and lets you cycle moods/reactions. *Gate:* preview shows a real fetched Petdex pet reacting; license note present.
5. **Selection → runtime** — define how a chosen pet reaches the live pet (e.g. `?pet=<id>` / `ANIMAYTE_PET`), reusing existing detection. *Gate:* picking a pet and reloading the pet view shows that pet. Confirm the mechanism with Saar.
